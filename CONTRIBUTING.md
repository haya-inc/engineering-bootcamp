# Contributing

このリポジトリは公開教材ですが、オープンソース教材ではありません。
貢献は歓迎しますが、利用と再配布は [LICENSE](LICENSE) の範囲に従います。

## 報告してほしいこと

Issueでは、次のような不備を報告できます。

- 誤字、表記ゆれ、分かりにくい説明
- リンク切れ、動画リンクの不一致
- 古くなった技術情報、現在の仕様と違う説明
- 課題READMEとstarterファイルの不整合
- `starter-apps/` の起動失敗、テスト失敗、再現できる不具合
- ライセンス、秘密情報、個人情報の扱いで誤解を招く記述

## Issueを書く前に

次を確認してください。

- 同じ内容のIssueがすでにないか。
- どのファイル、チャプター、行、リンクに関する報告か。
- starter appの不具合なら、OS、Node.jsのバージョン、実行したコマンド、エラー全文を整理したか。
- 課題に関する質問なら、解答そのものを公開Issueに貼っていないか。

## Issueに書いてよい情報

教材不備の報告では、次の情報があると確認しやすくなります。

```md
## 種別

誤字 / リンク切れ / 説明不足 / 古い技術情報 / starter app不具合 / その他

## 場所

ファイル:
チャプター:
該当箇所:

## 内容

何が問題か:

## 期待する状態

どうなっていると学習しやすいか:

## 確認したこと

実行したコマンド、見たページ、再現手順など:
```

## Issueに書いてはいけない情報

公開Issueには、次の情報を書かないでください。

- 課題の解答そのもの、完成した提出物の全文
- APIキー、パスワード、トークン、Cookie、秘密鍵
- 実在の顧客情報、個人情報、面談内容、個別の成績や評価
- 社外秘の仕様、未公開の事業情報、内部運用情報
- 他人の提出物、他人のアカウント情報、許可されていないログ

誤って秘密情報や個人情報を公開した場合は、Issueで詳細を広げず、READMEの「問い合わせと報告先」に従ってください。

## Pull Request

Pull Requestでは、誤字修正、リンク修正、説明の明確化、starter appの小さな修正を受け付けます。

PRには次を含めてください。

```md
## 変更内容

## 変更理由

## 対象ファイルまたはチャプター

## 確認したこと

## 秘密情報・個人情報チェック

- [ ] 秘密情報を含めていない
- [ ] 個人情報を含めていない
- [ ] 課題の解答そのものを公開していない
- [ ] LICENSEの範囲を広げる内容ではない
```

starter appを変更した場合は、該当アプリで `npm test` を実行し、結果を書いてください。

## 課題の解答を公開しない

この教材では、学習者が自分で観察、判断、検証結果を残すことを重視します。
そのため、IssueやPRに課題の解答例、完成済み提出物、採点に直結する詳細な回答を貼らないでください。

説明不足を報告するときは、「どこで迷ったか」「どの説明が足りないか」「どんな確認観点があるとよいか」を書いてください。

## ライセンス上の注意

このリポジトリの内容は、事前の書面許可なしに次の用途へ使えません。

- 再配布、転載、ミラー公開
- 商用研修、競合研修、コンサルティング、教材販売
- 派生教材、翻訳教材、スライド、動画、書籍、Webサイトとしての公開
- starter appの本番利用、顧客プロジェクト利用
- 機械学習モデル、データセット、検索コーパス、評価ベンチマークへの利用

詳細は [LICENSE](LICENSE) を確認してください。

## 行動方針

報告や議論では、学習者が安心して質問できる状態を保つことを重視します。
不備の指摘は歓迎しますが、個人の提出物、理解度、経験不足を責める形にしないでください。
