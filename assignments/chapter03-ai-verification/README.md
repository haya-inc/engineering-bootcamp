# 第3章 AI検証 ワークブック

このワークブックは、第3章の演習「AI利用計画を書く」と「AI利用ログを書く」で使う。

## 使い方

この資料を読み、`ai-use-plan.md` と `ai-use-log.md` を書く。目的は、AIをうまく使った雰囲気を出すことではなく、AIを何のために使い、何を検証し、どの提案を採用したのかを説明できるようにすることである。

## 前提

第2章で扱ったケースを使う。

> 研修の学習ログアプリに、メンター向け進捗一覧を追加したい。

第2章で作成した `decision-memo.md` を手元に置く。第3章では、このメモをもとにAIへ相談する。

## AIに相談してよいこと

例:

- 実装前に確認すべき不明点を洗い出す。
- 次に何を考えるべきかを提案してもらう。
- 作業計画、タスク分解、優先順位の候補を出してもらう。
- 範囲を決めて、調査や文書の下書きを任せる。
- 小さく区切ったコード修正案やテスト観点を出してもらう。
- 初回リリースで見落としやすいリスクを出す。
- Must / Should / Could / Won'tの整理を見直す。
- テスト観点の候補を出す。
- PR説明文や設計メモの初稿を作る。
- 調査すべき公式ドキュメントの候補を出す。

## AIの提案を確認なしに確定してはいけないこと

例:

- 採用案の最終決定
- 権限やセキュリティの安全性判断
- ライセンスや著作権の最終判断
- 本番データ、本番ログ、顧客情報を使った相談
- 実行してよいか分からないコマンドの承認
- 第2章で決めた目的や制約の上書き

AIには、答えだけでなく、次の作業や進め方も相談してよい。さらに、範囲を決めた調査、下書き、小さな修正案などの実作業も任せてよい。ただし、AIが出した作業計画や成果物は、目的、制約、リスクと照らして自分で確認する。

## 入力してはいけない情報チェック

AIに送る前に確認する。

```md
## 入力してはいけない情報

- [ ] APIキー、パスワード、トークンを含んでいない
- [ ] 個人情報、顧客情報を含んでいない
- [ ] 社外秘の仕様、未公開の事業情報を含んでいない
- [ ] 本番障害の詳細ログを含んでいない
- [ ] 契約書、法務確認中の文章を含んでいない
- [ ] ライセンスが不明な外部コードを含んでいない
- [ ] 会社や研修のAI利用ルールに反していない
```

## 良い依頼文の例

```txt
研修用の学習ログアプリに、メンター向け進捗一覧を追加する案を検討しています。

目的は、メンターが詰まっている受講者に早く気づけるようにすることです。
初回リリースでは、CSV出力、自動アラート、詳細な分析ダッシュボードは対象外にする予定です。

以下を出してください。

1. 実装前に確認すべき不明点
2. 次に考えるべき作業の順番
3. 初回リリースで見落としやすいリスク
4. 採用候補にしてよい案
5. 初回では見送るべき案
6. 採用候補にする案の設計メモ下書き
7. 公式ドキュメント、既存コード、テストで検証すべき点

不確かなことは断定せず、不明と書いてください。
```

```md

## 採用する提案

## 採用しない提案

採用しないものと理由:

| --- | --- | --- |

|    |    |    |

## 残っているリスク

確認が必要なこと:
```

## 検証方法の候補

| 検証したいこと | 検証方法 |
| --- | --- |
| APIや設定の使い方が正しいか | 公式ドキュメントを見る |
| 既存コードに同じ実装があるか | リポジトリ内検索をする |
| コードが動くか | ローカルで実行する |
| 既存挙動を壊していないか | テスト、lint、ビルドを実行する |
| セキュリティ上問題がないか | 権限、入力値、秘密情報の扱いを見る |
| 事業目的に合っているか | 第2章の `decision-memo.md` と照らす |
| 文章が正確か | 一次情報と照合する |

## 悪いログと良いログ

悪い例:

```md
AIに聞いたら、CSV出力がよいと言われた。
```

何が足りないか:

- 何を聞いたか分からない。
- 何を渡したか分からない。
- なぜCSV出力がよいのか分からない。
- 検証していない。
- 採用判断が第2章の目的とつながっていない。

良い例:

```md
## 目的

メンター向け進捗一覧の初回リリース範囲を検討する。

## AIに渡した情報

第2章のケース教材と、自分で作成した意思決定メモの要約。
個人情報、実在する顧客情報、秘密情報は渡していない。

## AIの出力要約

- CSV出力
- 未提出者フィルタ
- 自動アラート

## 採用する提案

未提出者フィルタは採用候補にする。メンターが詰まっている受講者に気づく目的に近いため。

## 採用しない提案

CSV出力と自動アラートは初回では見送る。CSVは利用目的が不明で、自動アラートは誤通知の運用リスクがあるため。

## 検証したこと

第2章の `decision-memo.md` と照らし、初回リリースでは一覧画面と必要最小限のフィルタに絞る方針と矛盾しないことを確認した。

## 未検証

閲覧権限の既存実装はまだ確認していない。
```

