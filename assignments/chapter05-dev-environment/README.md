# 第5章 開発環境 ワークブック

このワークブックは、第5章の演習「開発環境の現在地を確認する」「サンプルプロジェクトの構成を書く」「サンプルアプリを起動し、ログを読む」「よくあるエラーの次の確認を書く」で使う。

## 使い方

この資料を読み、`environment-check.md`、`project-overview.md`、`local-run-log.md`、`debug-note.md` を書く。

目的は、すべてのコマンドを覚えることではない。自分がどこで、何を実行し、何が起きたかを、あとから自分とメンターが再現できる形で残すことである。

## 前提

このリポジトリの `starter-apps/learning-log-sample` を使う。プロジェクト名は `learning-log-sample` である。

サンプルアプリの想定:

- 学習ログを記録する小さなWebアプリである。
- ローカルで開発サーバーを起動できる。
- ブラウザで `localhost` のURLを開いて確認できる。
- READMEにセットアップ手順が書かれている。
- 依存関係ファイル、ソースコード、テスト、設定ファイルが含まれている。

## 演習1: 開発環境の現在地を確認する

### 実行すること

次の観点を確認する。コマンド名は標準例であり、研修の標準環境に合わせて講師が調整してよい。

```txt
pwd
uname -a
which node
node --version
which npm
npm --version
which git
git --version
```

エディタのコマンドライン起動が設定されている場合:

```txt
code --version
```

設定されていない場合は、エディタの画面からバージョンを確認してもよい。

## 演習2: サンプルプロジェクトの構成を書く

### 実行すること

サンプルプロジェクトのルートに移動し、主要ファイルを確認する。

```txt
pwd
ls
```

`rg` が使える場合:

```txt
rg --files
```

`rg` が使えない場合は、エディタのファイルツリーや検索機能で確認してよい。

### 見るもの

- `README.md`
- 依存関係ファイル
- lockファイル
- `src/`

## 演習4: よくあるエラーの次の確認を書く

次のエラー例を読み、分類と次に確認することを書く。

### エラーA: コマンドが見つからない

```txt
zsh: command not found: npm
```

書くこと:

- 何が起きている可能性が高いか。
- どのコマンドで確認するか。
- 自分で判断できない場合、メンターに何を伝えるか。

### エラーB: 実行する場所が違う

```txt
npm ERR! enoent Could not read package.json
```

書くこと:

- 何が起きている可能性が高いか。
- 現在地をどう確認するか。
- プロジェクトルートへどう戻るか。

### エラーC: ポートが使われている

```txt
Error: listen EADDRINUSE: address already in use :::3000
```

書くこと:

- 何が起きている可能性が高いか。
- 別の開発サーバーが動いていないかどう確認するか。
- 勝手にプロセスを終了してよいか判断に迷う場合、何を相談するか。

### エラーD: 環境変数が足りない

```txt
Error: Missing required environment variable: DATABASE_URL
```

書くこと:

- 何が起きている可能性が高いか。
- `.env.example` をどう確認するか。
- 秘密情報をAIやチャットに貼らずに相談するには、何を伏せるか。

## AIに聞くときの例

```txt
研修用サンプルアプリのローカル起動で失敗しています。

実行した場所:
learning-log-sample

実行したコマンド:
npm run dev

期待結果:
http://localhost:3000 で画面が表示されること。

実際の結果:
起動に失敗しました。

ログ:
Error: listen EADDRINUSE: address already in use :::3000

出してほしいこと:
- 何が起きている可能性が高いか
- 次に確認すること
- 破壊的ではない確認方法

注意:
- 秘密情報は含めていません
- 不確かなことは断定しないでください
```
